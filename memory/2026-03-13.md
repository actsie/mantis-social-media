## Humanizer Skill Injection Fix — Proper Sub-Agent Implementation (8:30 PM PST)

**Root Cause Identified:** The `$(cat draft.txt)` shell expansion doesn't work in cron context. Sub-agents were being spawned but never received the actual draft text or skill instructions.

**Real Fix Implemented:**
1. **Read draft in parent script** — `const draftText = fs.readFileSync(...)`
2. **Read SKILL.md in parent script** — `const skillText = fs.readFileSync('skills/humanizer/SKILL.md', 'utf8')`
3. **Embed skill + draft in task** — Full 440-line skill instructions embedded directly in sub-agent prompt
4. **Spawn with workspace access** — `--cwd /Users/mantisclaw/.openclaw/workspace --mode run`
5. **Explicit save instruction** — Sub-agent told exactly where to write humanized output

**Why This Works:**
- No shell expansion needed (text embedded directly)
- Skill instructions ARE the task (can't skip them)
- Sub-agent has no choice but to follow skill (it's the entire prompt)
- Workspace cwd gives read/write access to draft files
